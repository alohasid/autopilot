# Autonomous Drone Mission: Point A to B (RC Override)

Цей проект містить Python-скрипт для керування БПЛА в симуляторі SITL за допомогою
бібліотеки Dronekit. Основна мета — виконання автономного польоту між двома точками в
режимі Stabilize з використанням механізму RC Override.

## 📋 Завдання (Task)

1. Встановлення середовища: Mission Planner та SITL симулятор.
2. Конфігурація вітру:
    SIM_WIND_SPD = 3
    SIM_WIND_DIR = 30
    SIM_WIND_TURB = 2
    SIM_WIND_TURB_FREQ = 0.
3. Місія:
    Зліт на висоту 100 м.
    Переліт з Точки А [50.450739, 30.461242] до Точки Б [50.443326, 30.448078].
    Режим польоту: Stabilize (утримання висоти та курсу через RC Override).
    Посадка: Ідеально в Точці Б.

## 🛠 Налаштування (Setup)

1. Встановлення залежностей
Переконайтеся, що у вас встановлений Python та Dronekit:

```
pip install dronekit
pip install pymavlink
```
2. Запуск SITL
Запустіть симулятор з вказанням координат старту (Точка А):

```
sim_vehicle.py -v ArduCopter -L 50.450739,30.461242,0,0 --console --map
```

3. Налаштування вітру
У консолі SITL або через Mission Planner введіть наступні команди:

```
param set SIM_WIND_SPD 3
param set SIM_WIND_DIR 30
param set SIM_WIND_TURB 2
param set SIM_WIND_TURB_FREQ 0.
```
## 🚀 Логіка роботи скрипта (Script Logic)

Оскільки режим Stabilize не передбачає автоматичної навігації за GPS-точками, скрипт
реалізує наступний алгоритм:

1. Зліт: Через simple_takeoff або емуляцію ручок газу (RC 3).
2. Навігація: Розрахунок азимуту (bearing) та дистанції до Точки Б.
3. RC Override:
    Канал 1 (Roll) та Канал 2 (Pitch) використовуються для корекції курсу.
    Канал 3 (Throttle) динамічно підтримує висоту 100 м.
    Канал 4 (Yaw) тримає ніс дрона в напрямку цілі.
4. Посадка: При досягненні радіусу < 2 м від Точки Б, скрипт ініціює плавне зниження
    газу до повної зупинки.
